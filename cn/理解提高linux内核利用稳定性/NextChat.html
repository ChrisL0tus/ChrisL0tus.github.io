To copy a directory and its contents using the cp command, you can use the -r or -R option, which stands for "recursive." Here's the syntax:

cp -r source_directory destination_directory

Here's a breakdown of the command:

  • cp: The command for copying files and directories.
  • -r or -R: The recursive option that allows copying directories and their contents.
  • source_directory: The directory you want to copy.
  • destination_directory: The directory where you want to paste the copied directory.

For example, if you want to copy a directory named "my_directory" located in the current directory to a destination directory named "backup," you would use the following command:

cp -r my_directory backup

This command will copy the "my_directory" directory and all its contents to the "backup" directory.

Note: If the destination directory doesn't exist, the cp command will create it. If the destination directory already exists, the copied directory will be placed inside it.

要使用ln -s命令创建符号链接（软链接），将路径链接到您想要的位置，可以按照以下语法进行操作：

ln -s target_path link_path

这里是命令的解释：

  • ln: 用于创建链接的命令。
  • -s: 创建符号链接（软链接）的选项。
  • target_path: 您要链接到的目标路径或文件。
  • link_path: 您要创建的链接路径或文件。

在您的情况下，假设您有一个名为compile_commands.json的文件，您希望将其链接到/path/to/desired_location/compile_commands.json，您可以使用以下命令：

ln -s /path/to/desired_location/compile_commands.json compile_commands.json

这将在当前目录中创建一个名为compile_commands.json的符号链接，该链接将指向/path/to/desired_location/compile_commands.json。

请确保在使用ln -s命令时提供正确的目标路径和链接路径。

当您通过SSH连接到服务器的终端时，如果长时间没有活动，会出现自动断开连接的情况。这是为了保护服务器和网络资源。为了保持SSH连接持久，您可以采取以下方法之一：

  1. 使用ssh命令的ServerAliveInterval参数：在连接时添加-o ServerAliveInterval=60选项，其中60表示每60秒发送一次保持活动的请求。这将告诉服务器保持与客户端的连接活动状态。
ssh -o ServerAliveInterval=60 user@server
  1. 在SSH服务器配置文件中进行配置：您可以在SSH服务器上的配置文件中设置ClientAliveInterval参数来保持连接活动。编辑SSH服务器的配置文件（通常是/etc/ssh/sshd_config），找到并修改以下行：
ClientAliveInterval 60
ClientAliveCountMax 3

将ClientAliveInterval设置为60表示每60秒发送一次保持活动的请求。ClientAliveCountMax表示如果在没有响应的情况下连续发送了3次保持活动的请求，服务器将断开连接。

  1. 使用终端多路复用工具：终端多路复用工具（如tmux或screen）可以创建持久会话，即使SSH连接断开，也可以在重新连接后恢复会话。您可以使用以下命令安装并使用tmux：
# 安装tmux
sudo apt-get install tmux

# 启动tmux会话
tmux

# 在tmux会话中运行您的脚本
your_script.sh

# 当连接断开后，重新连接到服务器并恢复tmux会话
tmux attach

这些方法中的任何一种都可以帮助您保持SSH连接持久。请根据您的需求选择适合您的方法。

在htop中，TIME+列显示的是进程的累计运行时间。它表示自进程启动以来，该进程在用户态和内核态下所消耗的总CPU时间。TIME+列是htop中的一个扩展列，它将用户态和内核态的时间相加，以提供更全面的进程运行时间信息。

通常，TIME+列显示的时间以秒为单位。该值可以帮助您确定进程在系统上执行的时间，以及它占用CPU资源的程度。较高的TIME+值表示进程已经运行了相当长的时间或者它在执行期间消耗了大量的CPU时间。

请注意，TIME+列不仅仅表示进程的实际执行时间，它还包括进程在等待CPU资源时的时间。因此，TIME+值可能会受到系统负载和其他因素的影响。

总结：htop中的TIME+列显示的是进程的累计运行时间，包括用户态和内核态下的CPU时间消耗。较高的值表示进程已经运行了相当长的时间或者消耗了大量的CPU时间。

您可以使用符号链接（Symbolic Link）来实现两个文件夹之间的链接。符号链接是一种特殊类型的文件，它包含指向另一个文件或目录的引用。通过创建一个符号链接，您可以让一个路径在执行时指向另一个路径。

在您的情况下，您可以创建一个符号链接，将/home/zhan4630/openwrt链接到/root/openwrt。这样，当代码中使用-I/home/zhan4630/openwrt/build_dir/target-i386_pentium4_musl/libwebsockets-full/libwebsockets-4.3.2/include路径时，实际上会访问/root/openwrt/build_dir/target-i386_pentium4_musl/libwebsockets-full/libwebsockets-4.3.2/include路径。

要创建符号链接，请使用ln命令，并使用-s选项指定创建一个符号链接。以下是创建符号链接的示例命令：

ln -s /root/openwrt /home/zhan4630/openwrt

这将在/home/zhan4630目录下创建一个名为openwrt的符号链接，它指向/root/openwrt目录。

请注意，创建符号链接需要足够的权限。您可能需要使用sudo命令以管理员权限执行上述命令。

创建符号链接后，当您的代码执行时，它将自动访问/root/openwrt目录下的文件。

请注意，符号链接是透明的，对于代码来说，它们看起来就像是普通的文件或目录。因此，您的代码不需要做任何修改，它将自动访问符号链接所指向的路径。
